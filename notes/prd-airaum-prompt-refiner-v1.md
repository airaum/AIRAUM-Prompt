# AIRAUM Prompt Refiner PRD v1.0

## 1. 제품 개요

### 제품명
AIRAUM Prompt Refiner

### 한 줄 설명
AIRAUM Prompt Refiner는 한국어·영어의 거친 사용자 입력을 범용 LLM이 더 잘 이해할 수 있는 구조적 단일 프롬프트로 정제하는 웹 도구입니다.

### 제품 유형
- 1페이지 웹 도구
- 텍스트 기반 프롬프트 정제기
- 무료 공개 가능한 정적 웹앱 우선

---

## 2. 문제 정의

적당히 AI를 사용해본 초중급 사용자들은 LLM에 질문을 넣을 때 다음 문제가 반복됩니다.

- 목적이 모호함
- 요청이 추상적임
- 출력 형식이 없음
- 불필요한 표현이 많음
- 모델이 해야 할 행동이 명확하지 않음

그 결과:
- 답변 품질 편차가 큼
- 같은 질문도 모델마다 다르게 해석됨
- 원하는 결과를 얻기 위해 여러 번 다시 입력해야 함

AIRAUM Prompt Refiner는 이 문제를 해결하기 위해, 사용자의 거친 입력을 복붙 가능한 단일 실행형 프롬프트로 정제합니다.

---

## 3. 목표

### 핵심 목표
- 초중급 사용자가 대충 입력한 요청을 범용 LLM 친화적 구조로 바꾼다.
- 결과는 사용자가 바로 복붙해서 사용할 수 있어야 한다.
- 출력은 너무 짧지 않되, 구조적으로 밀도 있는 단일 프롬프트여야 한다.
- 한국어/영어 사용자 모두 사용할 수 있어야 한다.
- 무료로 GitHub 및 Hugging Face에 공개 가능한 구조여야 한다.

### 제품 목표의 실무적 해석
- 답변 생성기가 아니라 프롬프트 정제기다.
- 전문가급 초정밀 튜닝보다 중간값 범용성을 우선한다.
- 장식적 문장보다 행동 지시와 출력 통제를 강화한다.

## 4. 비목표

- 유료 API 기반 서비스
- 사용자 계정/저장소/팀 협업 기능

## 5. 대상 사용자

### 주 사용자
- ChatGPT, Claude, Gemini 등 LLM을 어느 정도 사용해본 적은 있는 사용자
- 하지만 질문을 구조적으로 작성하는 습관이 약한 사용자
- 한국어 또는 영어 사용자
- 초중급 실사용자

### 비대상 사용자
- 모델별 프롬프트 튜닝을 깊게 다루는 전문가
- 장문 프롬프트 수동 설계에 익숙한 사용자
- 자동화 파이프라인 구축이 이미 끝난 사용자

## 6. 지원 언어

- 한국어
- 영어

### 언어 정책
- 입력 언어를 유지한다.
- 출력 언어도 입력 언어를 유지한다.
- 1차 버전에서는 혼합 다국어 최적화는 하지 않는다.

## 7. 입력 범위

### 허용 입력
- 텍스트만
- 질문형
- 요청형
- 설명형
- 1문장 ~ 짧은 문단 수준의 입력

### 제외 입력
- 파일 업로드
- 이미지
- 음성
- 외부 링크 분석
- 웹 검색 기반 보정
- 멀티턴 후속 질문 전제 입력

### 입력 특성
사용자 입력은 거칠고 추상적일 수 있다.
다만 시스템은 이를 가능한 범위에서 구조화하되, 과도한 상상으로 새로운 요구를 만들어내지 않는다.

---

## 8. 출력 범위

### 출력 원칙
- 결과는 최종 프롬프트 1개만 출력한다.
- 내부 분석 결과를 별도로 노출하지 않는다.
- 출력은 사용자가 즉시 복붙해서 쓸 수 있어야 한다.
- 출력은 지나치게 짧지 않되, 장식 문장으로 부풀리지 않는다.
- 출력은 구조적 밀도를 가져야 한다.

### 출력 스타일 원칙
최종 프롬프트는 가능한 한 다음 요소 중 다수를 포함한다.
- 해야 할 작업
- 행동 동사
- 제외 조건
- 출력 형식
- 필요 시 추론 또는 단계화 지시
- 필요 시 품질 기준 또는 주의 조건

---

## 9. 품질 기준

AIRAUM Prompt Refiner의 정제 품질은 아래 두 기준으로 평가한다.

1. LLM이 해야 할 행동이 동사 중심으로 재구성되었는가
2. 사용자가 바로 복붙해서 사용할 수 있는가

### 보조 기준
- 결과가 너무 빈약하지 않은가
- 결과가 기능 없는 장문으로 흐르지 않는가
- 범용 LLM에서 무난히 해석 가능한 중간값 구조인가

---

## 10. 안전 범위 및 제한

다음 요청은 정제 대상에서 제외하거나 제한한다.

- 명백한 불법행위를 촉진하는 요청
- 저작권 침해, 권리침해를 구체화하는 요청
- 사기, 해킹, 우회, 범죄 실행을 돕는 요청
- 법적 책임 회피를 목적으로 구체화하는 요청

### 처리 원칙
- 위험 요청은 정제하지 않는다.
- 필요 시 안전한 범위의 일반적 표현으로 제한하거나 거부한다.

## 13. 안내문 초안

### 이 도구가 잘 맞는 경우
- 질문이 막연해서 LLM 답변이 흐릴 때
- 복붙 가능한 프롬프트가 필요할 때
- 결과를 더 구조적으로 받고 싶을 때
- 한국어/영어 입력을 더 선명하게 정리하고 싶을 때

### 주의사항
- 이 도구는 답변 생성기가 아니라 프롬프트 정제기입니다.
- 전문가급 모델별 초정밀 튜닝은 목표가 아닙니다.
- 너무 짧거나 모호한 입력은 일반화된 보정이 들어갈 수 있습니다.
- 불법·권리침해·위험행위 목적 요청은 정제하지 않습니다.

### 제한사항
- 한국어/영어만 지원
- 텍스트만 지원
- 외부 검색 미연동
- 결과는 최종 프롬프트 1개만 출력

## 14. 정제 엔진 규칙

### 내부 처리 흐름
1. 입력 언어 판별
2. 핵심 의도 추출
3. 불필요 문장 제거
4. 행동 동사 중심 재구성
5. 구조 요소 삽입
6. 최종 프롬프트 단일 출력

### 규칙 설명

#### 1) 입력 언어 판별
- 한국어 또는 영어를 판별한다.
- 출력은 입력 언어를 유지한다.

#### 2) 핵심 의도 추출
입력에서 아래를 가능한 범위에서 추출한다.
- 사용자가 원하는 작업
- 원하는 결과
- 부족한 전제
- 문장 안의 핵심 목적

#### 3) 불필요 문장 제거
다음과 같은 기능 약한 표현은 제거 또는 약화한다.
- 좋게
- 잘
- 멋지게
- 대충
- 알아서
- 완벽하게

#### 4) 행동 동사 중심 재구성
최종 프롬프트는 가능한 한 행동 동사 중심으로 구성한다.
예:
- 분석하라
- 정리하라
- 분류하라
- 추론하라
- 제외하라
- 비교하라
- 단계화하라
- 출력하라

#### 5) 구조 요소 삽입
최종 프롬프트에는 필요에 따라 아래 요소를 포함한다.
- 작업 목표
- 처리 방식
- 제외 조건
- 출력 형식
- 주의 조건

#### 6) 단일 출력
사용자에게는 최종 프롬프트 한 덩어리만 보여준다.

## 15. 예외 입력 처리 규칙

### 유형 A. 너무 짧은 입력
예:
- 좋게 써줘
- 잘 정리해줘

처리 원칙:
- 가능한 범용 목적을 추정한다.
- 과도한 상상은 하지 않는다.
- 최소 실행형 프롬프트로 보정한다.

### 유형 B. 막연한 입력
예:
- 이거 좀 만들어보고 싶어
- 아이디어 정리해줘

처리 원칙:
- 목표 명확화 지시를 추가한다.
- 출력 형식을 보강한다.
- 불필요한 가정을 줄인다.

### 유형 C. 위험 요청
예:
- 침해, 우회, 범죄 실행 보조

처리 원칙:
- 정제하지 않음
- 안전 범위 내 일반적 표현만 허용하거나 거부

- 그러나 의미 없는 장문은 금지한다.

- 겉보기에 밀도 있어 보이되, 실사용성을 해치지 않아야 한다.

### 실무 해석
- 길이 자체가 목표가 아니다.
- 기능 밀도가 목표다.
- 공유/배포 시 보기 좋은 구조감은 허용하되, 반복과 장식은 금지한다.
